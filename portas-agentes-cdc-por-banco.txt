PORTAS PADRÃO DOS AGENTES CDC POR TIPO DE BANCO DE DADOS
IBM InfoSphere Data Replication
================================================================================

## Visão Geral
Este documento especifica as portas padrão utilizadas pelos agentes CDC 
(Capture e Apply) para cada tipo de banco de dados suportado.

USUÁRIOS DO SISTEMA OPERACIONAL PARA AGENTES CDC
================================================================================

Os agentes CDC (Capture e Apply) devem ser executados sob usuários dedicados
do sistema operacional para melhor segurança e gerenciamento.

┌─────────────────────────────────────────────────────────────────────────────┐
│  SERVIDOR SOURCE - Usuário para Capture Agent                              │
└─────────────────────────────────────────────────────────────────────────────┘

LINUX:
┌────────────────────────────────────────────────────────────────────────────┐
│  # Criar grupo                                                             │
│  groupadd cdcgroup                                                         │
│                                                                             │
│  # Criar usuário cdcuser                                                   │
│  useradd -m -g cdcgroup -s /bin/bash cdcuser                               │
│                                                                             │
│  # Definir senha                                                           │
│  passwd cdcuser                                                            │
│                                                                             │
│  # Criar diretórios necessários                                            │
│  mkdir -p /opt/ibm/cdc/capture                                             │
│  mkdir -p /staging/cdc                                                     │
│  mkdir -p /var/log/cdc                                                     │
│                                                                             │
│  # Definir permissões                                                      │
│  chown -R cdcuser:cdcgroup /opt/ibm/cdc                                    │
│  chown -R cdcuser:cdcgroup /staging/cdc                                    │
│  chown -R cdcuser:cdcgroup /var/log/cdc                                    │
│                                                                             │
│  # Permissões de diretório                                                 │
│  chmod 755 /opt/ibm/cdc/capture                                            │
│  chmod 755 /staging/cdc                                                    │
│  chmod 755 /var/log/cdc                                                    │
└────────────────────────────────────────────────────────────────────────────┘

WINDOWS:
┌────────────────────────────────────────────────────────────────────────────┐
│  # Criar usuário local (via GUI ou PowerShell)                             │
│  net user cdcuser <senha> /add /fullname:"CDC Capture User"                │
│                                                                             │
│  # Adicionar ao grupo Users                                                │
│  net localgroup Users cdcuser /add                                         │
│                                                                             │
│  # Criar diretórios necessários                                            │
│  mkdir C:\IBM\CDC\Capture                                                  │
│  mkdir D:\CDC\Staging                                                      │
│  mkdir D:\CDC\Logs                                                         │
│                                                                             │
│  # Definir permissões (via GUI ou icacls)                                  │
│  icacls C:\IBM\CDC\Capture /grant cdcuser:(OI)(CI)F                        │
│  icacls D:\CDC\Staging /grant cdcuser:(OI)(CI)F                            │
│  icacls D:\CDC\Logs /grant cdcuser:(OI)(CI)F                               │
└────────────────────────────────────────────────────────────────────────────┘

PERMISSÕES NECESSÁRIAS (Linux):
┌────────────────────────────────────────────────────────────────────────────┐
│  • Leitura de transaction logs do banco de dados                          │
│  • Escrita em diretórios de staging e logs                                │
│  • Execução de binários CDC                                               │
│  • Acesso de rede (portas 11001/11002/11003)                              │
│                                                                             │
│  Para DB2:                                                                 │
│  • Adicionar cdcuser ao grupo do DB2 instance owner (ex: db2iadm1)        │
│    usermod -a -G db2iadm1 cdcuser                                          │
│                                                                             │
│  Para PostgreSQL:                                                          │
│  • Adicionar cdcuser ao grupo postgres (se necessário)                    │
│    usermod -a -G postgres cdcuser                                          │
│                                                                             │
│  Para MySQL:                                                               │
│  • Adicionar cdcuser ao grupo mysql (se necessário)                       │
│    usermod -a -G mysql cdcuser                                             │
└────────────────────────────────────────────────────────────────────────────┘

================================================================================
CONFIGURAÇÃO DE LIMITES DO SISTEMA (LINUX)
================================================================================

Para garantir que os agentes CDC tenham recursos suficientes, configure os
limites do sistema operacional:

ARQUIVO: /etc/security/limits.conf
┌────────────────────────────────────────────────────────────────────────────┐
│  # Limites para usuário cdcuser                                            │
│  cdcuser soft nofile 65536                                                 │
│  cdcuser hard nofile 65536                                                 │
│  cdcuser soft nproc 16384                                                  │
│  cdcuser hard nproc 16384                                                  │
│  cdcuser soft stack 10240                                                  │
│  cdcuser hard stack 32768                                                  │
│  cdcuser soft memlock unlimited                                            │
│  cdcuser hard memlock unlimited                                            │
└────────────────────────────────────────────────────────────────────────────┘

VERIFICAR LIMITES:
┌────────────────────────────────────────────────────────────────────────────┐
│  # Logar como cdcuser e verificar                                          │
│  su - cdcuser                                                              │
│  ulimit -a                                                                 │
└────────────────────────────────────────────────────────────────────────────┘

│  # Executar instalador                                                     │

│  ./setup.bin                                                               │

│  • Selecionar "Capture Agent"                                              │

│  • Grupo: cdcgroup                                                         │
│  • Porta do agente: 11001 (DB2), 11002 (PG), 11003 (MySQL)                │
└────────────────────────────────────────────────────────────────────────────┘

INICIAR AGENTES CDC COMO SERVIÇO
================================================================================

LINUX (systemd):

CAPTURE AGENT - Criar arquivo de serviço:
┌────────────────────────────────────────────────────────────────────────────┐
│  # Criar arquivo /etc/systemd/system/cdc-capture.service                   │
│                                                                             │
│  [Unit]                                                                    │
│  Description=CDC Capture Agent                                             │
│  After=network.target                                                      │
│                                                                             │
│  [Service]                                                                 │
│  Type=forking                                                              │
│  User=cdcuser                                                              │
│  Group=cdcgroup                                                            │
│  ExecStart=/opt/ibm/cdc/capture/bin/dmts64 -I <instance_name>             │
│  ExecStop=/opt/ibm/cdc/capture/bin/dmshutdown -I <instance_name>          │
│  Restart=on-failure                                                        │
│  RestartSec=10                                                             │
│                                                                             │
│  [Install]                                                                 │
│  WantedBy=multi-user.target                                                │
│                                                                             │
│  # Habilitar e iniciar serviço                                             │
│  systemctl daemon-reload                                                   │
│  systemctl enable cdc-capture                                              │
│  systemctl start cdc-capture                                               │
│  systemctl status cdc-capture                                              │
└────────────────────────────────────────────────────────────────────────────┘

APPLY AGENT - Criar arquivo de serviço:
┌────────────────────────────────────────────────────────────────────────────┐

│                                                                             │
│  [Install]                                                                 │
│  WantedBy=multi-user.target                                                │

│  # Habilitar e iniciar serviço                                             │
│  systemctl daemon-reload                                                   │

WINDOWS (Serviço):
┌────────────────────────────────────────────────────────────────────────────┐

│  # Configurar para executar como usuário cdcuser:                         │
│                                                                             │
│  1. Abrir Services.msc                                                     │

MATRIZ DE CONECTIVIDADE COMPLETA
================================================================================

ORIGEM                          DESTINO                     PORTA      PROTOCOLO
─────────────────────────────────────────────────────────────────────────────
Usuário/Admin             →     Management Console          11701      HTTPS
Management Console        →     Access Server               10101      TCP
Access Server             →     Capture Agent (DB2)         11001      TCP
Access Server             →     Capture Agent (PostgreSQL)  11002      TCP
Access Server             →     Capture Agent (MySQL)       11003      TCP
Access Server             →     Apply Agent (DB2)           11101      TCP
Access Server             →     Apply Agent (PostgreSQL)    11102      TCP
Access Server             →     Apply Agent (MySQL)         11103      TCP
Capture Agent (DB2)       →     Apply Agent (DB2)           10001      TCP
Capture Agent (PostgreSQL)→     Apply Agent (PostgreSQL)    10002      TCP

================================================================================
OBSERVAÇÕES IMPORTANTES
================================================================================

1. PADRÃO DE NUMERAÇÃO:
   • Capture Agents: 1100X (onde X = 1 para DB2, 2 para PG, 3 para MySQL)
   • Apply Agents:   1110X (onde X = 1 para DB2, 2 para PG, 3 para MySQL)
   • Portas de dados CDC: 1000X (mesma lógica)

2. MÚLTIPLAS INSTÂNCIAS:
   Se você tiver múltiplas instâncias do mesmo tipo de banco, precisará usar
   portas diferentes para cada instância. Exemplo:
   • DB2 Instância 1: Capture 11001, Apply 11101
   • DB2 Instância 2: Capture 11011, Apply 11111

3. REPLICAÇÃO HETEROGÊNEA:
   Para replicação entre diferentes tipos de bancos (ex: DB2 → PostgreSQL):
   • Use a porta do Capture Agent correspondente ao banco source
   • Use a porta do Apply Agent correspondente ao banco target
   • Exemplo: DB2 (11001) → PostgreSQL (11102)

4. SEGURANÇA:
   • Considere usar SSL/TLS para comunicação entre componentes
   • Restrinja acesso às portas apenas aos IPs necessários
   • Use autenticação forte no Access Server
   • Monitore logs de acesso regularmente

5. PERFORMANCE:
   • Portas de dados CDC (100XX) devem ter baixa latência
   • Considere usar rede dedicada para replicação
   • Monitore throughput e ajuste buffers se necessário

6. TROUBLESHOOTING:
   • Se agente não inicia, verifique se porta não está em uso
   • Use logs do agente para diagnosticar problemas de conectividade
   • Verifique resolução de nomes (DNS ou hosts)
   • Confirme que firewalls permitem tráfego bidirecional
