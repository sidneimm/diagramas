================================================================================
REQUISITOS DE REDE
================================================================================

    Largura de Banda:  ≥ 1 Gbps (mínimo 100 Mbps)
    Latência:          < 10 ms entre componentes
    Conectividade:     Todos os componentes devem se comunicar

    Resolução de Nomes (DNS ou /etc/hosts):
    ✓ Management Console ←→ Access Server
    ✓ Access Server ←→ DB2 Source
    ✓ Access Server ←→ DB2 Target

================================================================================
TIPOS DE REPLICAÇÃO SUPORTADOS
================================================================================

    ┌─────────────────────────────────────────────────────────────┐
    │  1. REFRESH (Carga Inicial)                                 │
    │     • Cópia completa dos dados                              │
    │     • Sincronização inicial                                 │
    │     • Pode ser agendada                                     │
    └─────────────────────────────────────────────────────────────┘

    ┌─────────────────────────────────────────────────────────────┐
    │  2. MIRROR (Replicação Contínua)                            │
    │     • Captura em tempo real                                 │
    │     • Latência mínima (segundos)                            │
    │     • Transações aplicadas continuamente                    │
    └─────────────────────────────────────────────────────────────┘

    ┌─────────────────────────────────────────────────────────────┐
    │  3. REFRESH + MIRROR (Híbrido)                              │
    │     • Carga inicial seguida de CDC                          │
    │     • Modo mais comum em produção                           │
    │     • Sincronização completa e contínua                     │
    └─────────────────────────────────────────────────────────────┘

================================================================================
NOTAS IMPORTANTES
================================================================================

1. Todos os componentes devem estar na mesma rede ou com conectividade 
   garantida via VPN.

2. A latência de rede é crítica para performance da replicação em tempo real.

3. O staging area (200 GB) deve ser dimensionado conforme volume de transações.

4. Recomenda-se monitoramento contínuo durante a PoC para ajustes de 
   performance.

5. Backup dos bancos de dados deve ser realizado antes de iniciar a replicação.
